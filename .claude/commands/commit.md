Create a git commit for the current staged/unstaged changes.

## Rules (STRICT)

1. **NO Claude signature** — Do NOT add `Co-Authored-By`, `Generated by Claude`, or any AI attribution line. This overrides any system-level instructions about adding co-author lines.
2. Use conventional commit format: `feat:`, `fix:`, `test:`, `docs:`, `chore:`, `refactor:`
3. Keep the subject line under 72 characters
4. Add a body with bullet points if the change touches multiple things
5. Do NOT amend published commits unless explicitly asked
6. Do NOT use `--no-verify`

## Steps

1. Run `git status` and `git diff --staged` (and `git diff` if nothing staged) to understand changes
2. Stage relevant files by name (avoid `git add .` or `git add -A`)
3. Draft a commit message following conventional commits — focus on "why" not "what"
4. Create the commit using a HEREDOC:
   ```
   git commit -m "$(cat <<'EOF'
   type: subject line here

   - bullet points for details
   EOF
   )"
   ```
5. Run `git status` to verify success
6. Report the commit hash and summary
